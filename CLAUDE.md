# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

ANCMA is a collection of three independent demo web applications showcasing generative AI capabilities for ANCMA (Associazione Nazionale Ciclo Motociclo Accessori), the Italian motorcycle/scooter/cycling industry association. All content and UI is in Italian.

## Architecture

Three self-contained FastAPI apps, each with its own `main.py`, `requirements.txt`, templates, and data:

- **`app/`** — EICMA Network Visualizer. Builds a NetworkX graph of EICMA exhibitor relationships across 2022-2025 editions. Graph is built once at startup in `graph_builder.py` and served via `/graph` and `/stats` endpoints. Frontend is a static HTML file using D3.js force layout (`static/index.html`). No AI integration.

- **`policy-monitor/`** — Policy Monitor & Regulatory Intelligence. Displays regulations from `data/normative.json` with filtering. The `/api/analyze/{norm_id}` endpoint calls Claude API to generate structured JSON analyses (impact assessment, timeline, recommendations). Analyses are cached as individual JSON files in `data/analyses/`. Frontend served via Jinja2 template.

- **`dashboard-mercato/`** — Dashboard Mercato Moto Italia. Market registration data (2022-2025) hardcoded in `main.py` as Python dicts. D3.js v7 frontend with KPI cards, charts, and an AI chatbot (`/api/chat`) that uses Claude API with the full dataset injected into the system prompt.

## Running Locally

Each app runs independently from its own directory:

```bash
# EICMA Network (no API key needed)
cd app
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
python main.py                    # http://localhost:8000

# Policy Monitor (needs ANTHROPIC_API_KEY for AI analysis)
cd policy-monitor
pip install -r requirements.txt
python main.py                    # http://localhost:8001

# Dashboard Mercato (needs ANTHROPIC_API_KEY for chatbot)
cd dashboard-mercato
pip install -r requirements.txt
python main.py                    # http://localhost:8002
```

Set `ANTHROPIC_API_KEY` in a `.env` file or environment variable for apps that use Claude API.

## Deployment

All three apps deploy to Render (free tier, Frankfurt region) via `render.yaml` at repo root. Each service maps to its `rootDir` and runs `uvicorn main:app --host 0.0.0.0 --port $PORT`. The `PORT` env var is provided by Render.

## Key Patterns

- Each app is a single `main.py` file with all routes — no module splitting.
- Frontends are single HTML files with inline CSS/JS (no build step, no bundler).
- Data is either hardcoded in Python (dashboard-mercato), loaded from JSON files on disk (app, policy-monitor), or generated by Claude API and cached to disk (policy-monitor analyses).
- Claude API calls use `claude-sonnet-4-20250514` model via the `anthropic` Python SDK.
- The EICMA app uses `python-louvain` for community detection with a `networkx` fallback.
- JSON data files at repo root (`espositori_eicma_*.json`, `ancma_associati.json`, `ancma_eicma_matched.json`) are source data; copies in `app/data/` are what the app actually reads.
